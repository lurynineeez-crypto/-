# 智能种植方舱系统 UI 标准与页面壳规范

本文档用于约束后续全站 UI 改造。首页是视觉母版，但不是所有页面都要做成首页大屏。后续改页面时，先选择页面壳，再调用 token 和组件 class，避免每个页面单独写一套颜色、字号、按钮和卡片。

## 1. 设计目标

- 所有页面都属于同一个“智能种植方舱数字孪生系统”。
- 首页保持最强的深绿数字孪生总控感。
- 高频操作页继承首页的深绿科技感，但要更适合任务执行、报警处理和巡检操作。
- 低频管理页保持企业系统的阅读和管理效率，不强行大屏化。
- UI 统一只改变视觉语言和组件表达，不改变业务字段、mock 数据结构、接口、路由和 3D 加载逻辑。

## 2. 页面壳分层

### 2.1 `.twin-page`

适用页面：

- 总览 / 3D 数字孪生总控台

视觉特征：

- 深绿、墨绿、深青绿色背景。
- 中央 3D 舞台是首屏主角。
- 左右面板承载任务、环境、风险、流程。
- 底部 command bar 承载系统状态、点位统计、快捷入口。
- glow 可以存在，但必须克制，服务于“数字孪生总控”而不是装饰。

排版原则：

- 主标题必须强，但不要压过 3D 舞台。
- 面板标题统一 16px 左右。
- 关键数字在 24-28px 区间，只有总控级大数字可到 40px。
- 辅助说明控制在 10-12px，不能成为主要阅读负担。

### 2.2 `.ops-page`

适用页面：

- 今日作业
- SOP 工单
- 报警处置
- 环境监测
- 点位巡检 / 三维巡检

视觉特征：

- 深绿智能操作工作台。
- 背景与首页同源，但布局更像工作台，不像展厅大屏。
- 任务卡、风险卡、流程节点、事件表是主角。
- 信息密度高于首页，按钮和状态必须清楚可执行。
- 不使用大面积白底，不使用过强 glow。

排版原则：

- 页面标题 20-22px。
- 操作页关键数字 20-28px。
- 任务标题 14-16px。
- 字段标签 11-12px。
- 正文与说明 13-14px。
- 表格和事件列表 12-13px。

### 2.3 `.app-page`

适用页面：

- 种植批次
- 阶段管理
- 菌包状态
- 采收质量
- 报告文件
- 审计追溯
- 工艺配方
- 种植日历
- 清舱消毒
- 生物安全记录
- 质量放行

视觉特征：

- 企业系统阅读页 / 管理页。
- 可以使用浅绿灰背景或轻深色壳。
- 统一字体、按钮、卡片、表格、状态色。
- 不强行使用首页大屏布局。

排版原则：

- 页面标题 20-22px。
- 面板标题 16-18px。
- 卡片标题 14-16px。
- 正文 13-14px。
- 表格 12-13px。
- 状态标签 11-12px。

## 3. Design Token

### 3.1 字体

全站统一：

```css
font-family: Inter, ui-sans-serif, system-ui, -apple-system, BlinkMacSystemFont, "Segoe UI", "PingFang SC", "Microsoft YaHei", sans-serif;
```

不要在业务页单独引入 display font。产品 UI 以稳定、清晰、可读为第一优先级。

### 3.2 首页数字孪生 token

当前已在 `styles.css` 中建立：

```css
--twin-font-risk-title: 18px;
--twin-font-risk-title-compact: 16px;
--twin-font-card-title: 13px;
--twin-font-body: 12px;
--twin-font-meta: 11px;
--twin-font-caption: 10px;
--twin-font-micro: 9px;
--twin-font-micro-compact: 8px;
--twin-font-number-lg: 28px;
--twin-font-number-lg-compact: 26px;
--twin-font-number-md: 24px;
--twin-font-number-md-compact: 22px;
--twin-font-number-sm: 19px;
--twin-font-number-sm-compact: 17px;

```

用途：

- `title`：首页系统主标题。
- `subtitle`：首页英文副标题、系统短说明。
- `panel-title`：左右面板标题、中央舞台标题。
- `risk-title`：最高风险卡标题。
- `card-title`：任务卡标题、环境数据项标题。
- `body`：普通正文。
- `meta`：字段说明、状态辅助文字。
- `caption`：标签、表头、短状态。
- `micro`：极紧凑信息，如流程节点短说明。
- `number-*`：风险值、底部状态数字、环境读数。

## 4. 基础组件规范

### 4.1 页面标题区 `.app-page-header` / `.ops-page-header`

必须展示：

- 页面名称。
- 当前批次或当前上下文。
- 当前阶段或当前处理状态。
- 关键计数，如今日任务、待处理、待复核、待关闭。

不要展示：

- 长段说明文案。
- 重复解释系统能力的营销文案。
- 与当前任务无关的装饰数字。

### 4.2 状态 chip `.app-status-chip`

用途：

- 正常、预警、报警、离线、在线。
- 待处理、处理中、待复核、已完成、已关闭。

规则：

- 状态文案必须中文。
- 不出现英文 `warning`、`pending`、`done` 作为界面文字。
- chip 高度保持紧凑，优先 24-28px。

### 4.3 任务卡 `.app-task-card` / `.ops-task-card`

必须包含：

- 优先级 P1/P2/P3。
- 任务标题。
- 截止时间。
- 关联点位、设备或批次。
- 当前状态。
- 责任人。
- 复核人。
- 判断依据。
- 标准动作。
- 留档材料。
- 主操作按钮。

视觉规则：

- P1 使用琥珀强调。
- P2 使用青绿或弱琥珀强调。
- P3 使用弱青绿，不能像禁用。
- 操作页任务卡使用深色半透明卡片。
- 管理页任务卡可以使用浅底，但必须使用统一边框、标签和按钮。

### 4.4 风险卡 `.app-risk-card` / `.ops-risk-card`

必须包含：

- 风险名称。
- 当前值。
- 目标值或阈值。
- 偏差。
- 点位。
- 持续时长。
- 责任人。
- 复核人。
- 建议操作。
- 操作入口。

视觉规则：

- 风险主色使用琥珀。
- 报警级别可使用红色，但不要整卡大面积红色。
- 风险值使用 `number-lg` 或操作页关键数字字号。

### 4.5 流程节点 `.app-workflow-step`

首页 / 操作页流程：

- 报警 -> 分派 -> 处理 -> 复核 -> 关闭。
- 看 -> 判 -> 做 -> 核 -> 留。

状态规则：

- 已完成：青绿。
- 当前：琥珀。
- 待处理：低对比弱化。
- 异常 / 阻塞：红色。

不要把流程节点做成普通浅色小卡片。操作页应使用深色轨道和连接线。

### 4.6 数据面板 `.app-data-panel`

适用：

- 环境数据。
- 点位详情。
- 批次阶段摘要。
- 报告统计。

规则：

- 面板标题 16-18px。
- 数据名称 12-13px。
- 数据值 19-28px，根据页面层级选择。
- 单位不能比数值更抢眼。
- 数据面板内的说明文字不要超过两行。

### 4.7 表格 `.app-table` / `.ops-table`

规则：

- 表头 12px，弱文字。
- 表格正文 12-13px。
- 状态列使用统一 chip。
- 操作列使用统一按钮，不裸露普通文本链接。
- 操作页可使用深色事件表。
- 管理页可使用浅底表格，但表头、边框、hover 必须统一。

### 4.8 按钮

主按钮 `.app-button-primary`：

- 用于立即复核、开始处理、提交复核、生成报告等主动作。
- 操作页用深绿/青绿或琥珀高亮。

次按钮 `.app-button-secondary`：

- 用于查看详情、取消、返回、筛选等次动作。

风险按钮 `.app-button-warning`：

- 用于分派、进入报警处置、处理风险。
- 使用琥珀，不使用大面积红色。

按钮规则：

- 文案必须明确动作，如“立即复核”“提交复核”“开始处理”“查看详情”。
- 不使用“操作”“确认”这类含糊文案。
- 同一页面主按钮不要超过 1-2 个视觉主色。

## 5. 数据排版规则

### 5.1 标题层级

- 系统主标题：24-28px。
- 普通页面标题：20-22px。
- 面板标题：16-18px。
- 卡片 / 任务标题：14-16px。
- 首页紧凑任务标题：13px。
- 正文：13-14px。
- 首页紧凑正文：12px。
- 辅助说明：11-12px。
- 极紧凑说明：9-10px。

### 5.2 数字层级

- 首页关键数字：24-40px。
- 首页风险值：28px。
- 首页底部状态数字：24px。
- 首页环境读数：19px。
- 操作页关键数字：20-28px。
- 管理页指标数字：18-24px。

### 5.3 信息组织

任务类信息按以下顺序：

1. 优先级 / 状态。
2. 任务标题。
3. 关联对象和截止时间。
4. 判断依据。
5. 标准动作。
6. 留档材料。
7. 操作按钮。

报警类信息按以下顺序：

1. 报警名称和风险值。
2. 点位、当前值、目标值、偏差。
3. 持续时长。
4. 责任人和复核人。
5. 建议操作。
6. 闭环流程。
7. 队列和操作入口。

报告 / 批次类信息按以下顺序：

1. 当前状态。
2. 当前阶段或报告类型。
3. 关键时间。
4. 责任人。
5. 证据或附件入口。
6. 查看 / 导出 / 追溯按钮。

## 6. 页面改造执行顺序

### P0

- 今日作业：必须是深绿操作工作台。
- 报警处置：必须是深绿闭环处理台。

目标：

- 证明系统能指导种植员干活。
- 证明报警能从发现、分派、处理、复核到关闭。

### P1

- 三维巡检。
- 环境监测。
- 点位巡检。

目标：

- 证明点位、环境和三维巡检能力专业可信。
- 使用 `.ops-page`，但不做成首页大屏。

### P2

- 种植批次。
- 阶段管理。
- 菌包状态。
- 采收质量。

目标：

- 统一卡片、状态、表格和按钮。
- 使用 `.app-page`，不大改业务结构。

### P3

- 报告文件。
- 审计追溯。
- 工艺配方。
- 种植日历。
- 清舱消毒。
- 生物安全记录。

目标：

- 轻量统一视觉。
- 保持阅读和交付效率。

## 7. 页面接入检查表

每次改一个页面前，必须回答：

- 这个页面属于 `.twin-page`、`.ops-page` 还是 `.app-page`？
- 页面首屏主任务是什么？
- 主按钮是什么？
- 状态文案是否全中文？
- 是否使用统一 tag / chip / button？
- 是否新增了本页私有颜色？如果新增，是否真的必要？
- 是否出现大面积白底和首页割裂？
- 是否把低频管理页强行做成大屏？
- 是否改动了 mock 数据结构、接口、路由或 3D？
- 是否运行构建验证？

## 8. 禁止事项

- 不改 mock 数据结构。
- 不改接口。
- 不改路由结构。
- 不改 3D 模型文件。
- 不改 GLTFLoader。
- 不改 CabinScene 逻辑。
- 不新增 UI 库。
- 不新增页面。
- 不把所有页面都做成首页大屏。
- 不在每个页面单独写一套按钮、标签、颜色。
- 不出现英文状态词作为界面文案。
- 不使用大面积纯白后台外壳。

## 9. 验收标准

- 从首页切到今日作业，仍然像同一个深绿智能系统。
- 从首页切到报警处置，仍然像同一个深绿智能系统。
- 操作页比首页更适合执行任务和处理事件。
- 管理页比首页更适合阅读、查询、交付和追溯。
- 字号层级稳定，不同页面同类信息大小一致。
- 按钮、标签、状态色、卡片边框统一。
- 首页 3D 不受影响。
- mock 数据结构不受影响。
- 构建通过。
