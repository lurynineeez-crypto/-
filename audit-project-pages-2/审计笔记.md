# 种植方舱运营工作台审计笔记

## 审计范围

- 审计对象：本地前端 `http://localhost:5173/`
- 审计目的：检查整个项目页面是否按用户使用路径收拢，找出仍然杂乱、重复和不清晰的地方，并给出重排建议
- 证据来源：本次审计中保存的本地截图
  - `01-overview-home.png`
  - `02-overview-brain.png`
  - `03-overview-cabin.png`
  - `04-overview-screen.png`

## 用户目标和可访问性目标

- 用户进入系统后，先快速知道当前批次、环境风险、报警和下一步该做什么
- 进入各业务页面时，路径要短、分类要稳定，不要在相似页面之间来回跳
- 关键数据要能一眼看懂，不能靠猜标题
- 大屏、工作台和辅助页要各自有明确角色，避免所有页面都堆成“同级菜单”

## 这版的优点

1. 顶部壳子已经统一，角色、批次、搜索、在线状态和主导航都在一个稳定框架里。
2. 首页工作台的中心非常清楚，3D 方舱、当前环境、报警、建议和设备状态形成了一个完整闭环。
3. 三个首页态已经有分工：
   - `首页驾驶舱`：偏日常监控和处置
   - `总控大脑`：偏决策与复盘
   - `三维方舱`：偏空间和点位理解
   - `独立大屏`：偏展示和汇报
4. 页面风格比前一轮干净了很多，基础视觉秩序已经成立。

## 还乱的地方

### 1. 同类功能还是分散成太多并列页

现在虽然顶层已经分成 7 组，但组内仍然有大量相似页面并列存在，用户还是要自己判断该进哪一个：

- 种植生命周期里：批次、生长、生产、采收、清舱、经验、复盘、质量
- 生产决策里：主任、决策、经营、协同、策略、专业分析
- 运维设备里：环境、控制、报警、维护、点位
- 数据与交付里：报表、审计、追溯、历史、配置、治理、工程、联调、验收、演示、交付、演练
- 系统支持里：登录、设置、演示、搜索、解释、多端通知

这会让系统看起来“功能很多”，但实际操作路径还是偏散。

### 2. 大屏页的角色过强，但信息表达还不够纯

`04-overview-screen.png` 里的大屏很醒目，但标题区仍然有明显乱码/旧文本残留，说明这类页面还没完全收口。

而且大屏现在更像“另一套页面”，不是首页的延伸，这会让用户理解成本变高。

### 3. 首页中心区偏重，阅读负担有点高

`01-overview-home.png` / `03-overview-cabin.png` 的中间 3D 视图很强，但上下左右还有很多摘要、报警、建议、点位和设备块。

问题不是信息少，而是信息同时抢注意力，用户要自己找重点。

### 4. 组内页面的重复感还在

有些页面本质是在做同一件事，只是换了标题和切口：

- 复盘和经验沉淀很接近
- 质量、经营、专业分析之间有明显重叠
- 报表、审计、追溯、历史查询之间边界不够硬
- 报警、维护、控制、环境联动之间也容易互相覆盖

这会造成“功能越加越多，路径却没有更顺”。

## 可见的可访问性风险

1. `04-overview-screen.png` 中的大屏标题区文本异常，属于明显的可读性问题，用户会直接怀疑系统稳定性。
2. 右侧报警卡片和中部点位区的信息密度较高，小字号内容比较依赖视觉扫描，弱视用户会更吃力。
3. 当前页面主要靠颜色和卡片层级表达状态，后续还要补更明确的状态文案和可点击反馈。
4. 仅凭截图不能确认键盘操作、焦点顺序、读屏语义、缩放后的断行是否正常，这些都还需要单独测。

## 建议的用户路径重排

我建议把整个系统收成 4 条主路径，而不是继续堆同级页面：

1. `总控首页`
   - 首页驾驶舱
   - 总控大脑
   - 三维方舱
   - 独立大屏

2. `日常作业`
   - 任务协同
   - 生命周期相关的日历、工单、SOP、巡检

3. `种植闭环`
   - 批次管理
   - 生长阶段
   - 生产与采收
   - 质量复盘
   - 清舱复盘
   - 经验优化

4. `决策与运维`
   - 生产决策
   - 环境监测
   - 联动控制
   - 报警事件
   - 维护校准
   - 点位档案

5. `数据与交付`
   - 报表、审计、追溯、历史
   - 配置、治理
   - 工程交付、联调文档、验收、演示

6. `系统支持`
   - 登录权限
   - 系统设置
   - 全局检索
   - 多端通知
   - 解释自检

这样改完以后，用户会先看“我现在在什么阶段”，再看“我要处理什么”，最后才进“数据和交付”。

## 结论

- 这版已经从“散页面堆叠”进到了“可用工作台”的阶段
- 真正还需要收口的，不是功能数量，而是功能归属和页面角色
- 下一轮最值得做的是：
  1. 把同类页面合并到更少的工作台里
  2. 把大屏页改成首页的一个展示模式
  3. 把乱码和旧标题清理干净
  4. 把首页中心区再减负，让主线更清晰
